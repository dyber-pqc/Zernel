# Contributing to Zernel

Thank you for your interest in contributing to Zernel.

## Development Setup

```bash
./scripts/setup-dev-env.sh
```

## Running Tests

```bash
./scripts/test-all.sh
```

## Code Style

- Rust: `cargo fmt` and `cargo clippy` must pass
- All kernel components (scheduler, eBPF) are GPL-2.0
- CLI and enterprise features are proprietary

## Pull Request Process

1. Fork the repository
2. Create a feature branch
3. Ensure all tests pass
4. Submit a pull request
